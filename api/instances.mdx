## Overview

Instances represent a user's active enrollment in a study plan. Each instance has its own schedule, reminders, and tracks completion independently. When created, instances snapshot the study plan's resources at that moment.

## List Instances

<CodeGroup>
```bash cURL
curl -X GET "https://your-domain.com/api/instances?status=active" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch('/api/instances?status=active', {
  headers: { 'Authorization': `Bearer ${token}` }
});
const data = await response.json();
```
</CodeGroup>

`GET /api/instances`

Get user's study plan instances. Requires authentication.

<ParamField query="status" type="string">

## Create Instance

<CodeGroup>
```bash cURL
curl -X POST "https://your-domain.com/api/instances" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "studyPlanId": "507f1f77bcf86cd799439011",
    "startDate": "2026-03-10T00:00:00Z",
    "endDate": "2026-04-10T00:00:00Z",
    "reminderTime": "09:00",
    "resourceSchedule": []
  }'
```

```javascript JavaScript
const response = await fetch('/api/instances', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    studyPlanId: '507f1f77bcf86cd799439011',
    startDate: '2026-03-10T00:00:00Z',
    endDate: '2026-04-10T00:00:00Z',
    reminderTime: '09:00'
  })
});
```
</CodeGroup>

`POST /api/instances`

Create a new study plan instance. Snapshots the plan's resources at creation time.

### Request Body

<ParamField body="studyPlanId" type="string" required>
  Study plan ID to create instance from
</ParamField>

<ParamField body="startDate" type="string" required>
  Start date (ISO 8601 format)
</ParamField>

<ParamField body="endDate" type="string" required>
  End date/deadline (ISO 8601 format). Must be after startDate.
</ParamField>

<ParamField body="reminderTime" type="string" default="09:00">
  Daily reminder time in HH:MM format
</ParamField>

<ParamField body="resourceSchedule" type="array">
  Optional resource scheduling information
</ParamField>

### Response

<ResponseField name="message" type="string">
  "Instance created successfully"
</ResponseField>

<ResponseField name="instance" type="object">
  The created instance object
</ResponseField>

## Update Instance

<CodeGroup>
```bash cURL
curl -X PUT "https://your-domain.com/api/instances/507f1f77bcf86cd799439011" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "completed",
    "notes": "Finished ahead of schedule!",
    "reminderEnabled": false
  }'
```
</CodeGroup>

`PUT /api/instances/:id`

Update an instance. Only the owner can update.

### Path Parameters

<ParamField path="id" type="string" required>
  Instance ID
</ParamField>

### Request Body

All fields are optional. Only include fields to update.

<ParamField body="startDate" type="string">
  Start date (ISO 8601)
</ParamField>

<ParamField body="endDate" type="string">
  End date (ISO 8601)
</ParamField>

<ParamField body="status" type="string">
  Status: `active`, `completed`, or `paused`
</ParamField>

<ParamField body="progress" type="number">
  Progress percentage (0-100)
</ParamField>

<ParamField body="reminderTime" type="string">
  Reminder time (HH:MM)
</ParamField>

<ParamField body="reminderEnabled" type="boolean">
  Enable/disable reminders
</ParamField>

<ParamField body="customReminders" type="array">
  Custom reminder schedule
</ParamField>

<ParamField body="notes" type="string">
  Instance notes
</ParamField>

<ParamField body="resourceNotes" type="object">
  Per-resource notes as key-value pairs: `{ resourceId: "note text" }`
</ParamField>

<ParamField body="customTitles" type="object">
  Custom resource titles
</ParamField>

<ParamField body="resourceIds" type="array">
  Reorder/modify instance resources (uses snapshotResourceIds)
</ParamField>

### Response

<ResponseField name="message" type="string">
  "Instance updated successfully"
</ResponseField>

<ResponseField name="instance" type="object">
  Updated instance object
</ResponseField>

## Key Concepts

### Resource Snapshotting

When an instance is created, it snapshots the study plan's resources at that moment:

- Stored in `snapshotResourceIds` field
- Independent of future study plan changes
- Ensures instance stability even if plan is modified

### Progress Tracking

Progress is tracked globally per user per resource:

- Completing a resource in one instance marks it complete everywhere
- Use the [User Progress API](/api/user-progress) to manage completion status
- Instance tracks which resources are completed via `completedResources` array

### Automatic Updates

The API automatically calculates:

- `totalTime` and `completedTime` based on resource metadata
- `resourcePercent` and `timePercent` for progress tracking
- `remainingTime` for scheduling
- `lastAccessedAt` when instance is viewed
